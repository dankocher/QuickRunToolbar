Getting Started

Quick Run Toolbar lets you run your Run/Debug Configurations with a single click from the Main Toolbar. Follow these steps to get started:

1) Requirements

  • Install a compatible JetBrains IDE, such as IntelliJ IDEA (Community or Ultimate) or any IntelliJ‑based IDE, version 2024.1 or newer.
  • Ensure your project has the Run/Debug Configurations you want to use.

2) Install the plugin

  1. Launch the IDE and open Settings/Preferences > Plugins.
  2. In the Marketplace tab, search for Quick Run Toolbar and click Install.
  3. Restart the IDE when prompted.

3) Show the Main Toolbar (if hidden)

Go to View > Appearance > Toolbar and make sure it is enabled.

4) Configure your buttons

  1. Open Tools > Quick Run: Edit Buttons.
  2. In the editor:
    • Reorder: drag rows to change the button order.
    • Icon: click the Icon cell to pick an icon (built‑in search and Choose from file… for SVG/PNG).
    • Name: edit the Name column to set the button label (optional).
    • Show Name: toggle to show/hide the text label next to the icon.
    • Enabled: toggle each button on/off without changing the original configuration.
  3. Click OK to save. The Main Toolbar updates immediately.

5) Run

  • Click a toolbar button to run the corresponding configuration with the Run executor.
  • If labels are enabled, they appear next to the icon; otherwise buttons are icon‑only.

Notes & Troubleshooting

  • Only existing, non‑temporary Run/Debug configurations are listed.
  • If a configuration is removed from the project, its button disappears automatically.
  • If you don’t see the buttons, ensure the toolbar is visible and verify the desired items are enabled in Edit Buttons.

Complementary or conflicting plugins

  • Complementary: works with any plugin that adds Run/Debug configuration types; no special integration required.
  • Conflicts: no known technical conflicts. There may be functional overlap with other plugins that add run buttons to the Main Toolbar; if using multiple, you may prefer to keep only one to avoid duplicated controls.
